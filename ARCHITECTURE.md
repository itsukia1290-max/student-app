# Architecture Notes

このファイルはリポジトリの構成方針と今後の整理提案を記載しています。

目的
- 新しい開発者が素早く構成を理解できるように、機能ごとの責務を明示する。
- 今後のリファクタリングやファイル移動で破壊的な変更を避けるためのガイドラインを提供する。

推奨ディレクトリ構成（段階的移行）

- `src/`
  - `pages/` : ルーティングやページ単位のコンテナコンポーネント（画面ごとのロジック）
  - `components/` : 再利用可能な UI コンポーネント（小〜中サイズ）。更に `components/ui/` を作成してボタン・入力・カード等を集約する。
  - `contexts/` : React Context（Auth 等）
  - `hooks/` : カスタムフック
  - `lib/` : 外部クライアント（supabase 初期化など）
  - `utils/` : 純粋関数ユーティリティ（画像/ストレージラッパー等）

移行計画
1. まずは「ドキュメント化」と「責務コメントの追加」を行う（今回の変更）。
2. 次に `components/ui/` を作成し共通ボタン/入力/Card を移動、既存コンポーネントを順次 wrapper に置換。
3. ページごとに `pages/<PageName>/index.tsx` + `pages/<PageName>/styles.css` のように分ける（必要なら）。
4. 大きな移動は PR を分割して段階的に行う（各 PR はビルド・テストを通すこと）。

ドキュメントは簡潔に、実際のコード変更は段階的に行うことを推奨します。
